FoodSynk - Bitacora del bloque CRUD de Recetas

Documento de seguimiento desde el momento en que empezamos el CRUD de recetas (backend Laravel + Orion), incluyendo comandos, sintaxis, objetivos y resultados reales.

1) Punto de partida

  • Tablas principales y pivotes ya creadas: recipes, ingredients, categories, ingredient_recipe, category_recipe.
  • Seguridad base ya trabajada antes del CRUD funcional: Policy owner-only, validaciones, orion.use_validated=true, batch desactivado.
  • Objetivo del bloque: que store/update de recetas guarde datos y sincronice relaciones.

2) Comandos clave y sintaxis

Comando Para que sirve Sintaxis explicada
php artisan make:request NombreRequest Crear un FormRequest para validacion. make:request genera clase en app/Http/Requests.
php artisan make:migration nombre --table=tabla Crear migracion de ajuste sobre tabla existente. --table indica que es alteracion, no creacion inicial.
php artisan migrate Aplicar migraciones pendientes. Ejecuta metodo up() de cada migracion no corrida.
php -l ruta/archivo.php Verificar sintaxis PHP (lint). -l = lint. No ejecuta logica de app ni toca BD.
php artisan tinker --execute="..." Probar consultas/modelos rapido. Ejecuta una expresion PHP dentro del contexto Laravel.
curl.exe -X POST/PATCH ... Probar endpoints API. -X metodo HTTP, -H headers, -d body JSON.

3) Paso a paso ejecutado (con resultados esperados vs reales)

Paso A - Integrar sync de categorias e ingredientes en controlador

  • Archivo tocado: app/Http/Controllers/Api/RecipeController.php.
  • Se implemento afterSave(...) para hacer categories()->sync(...) y ingredients()->sync(...).
  • Resultado esperado: guardar receta y sincronizar pivotes en store/update.

Paso B - Primera prueba API de create y primeros errores

  • Comando de prueba: curl.exe -X POST /api/recipes ....
  • Error real: Unknown column 'name' in ingredients al crear ingrediente de prueba.
  • Diagnostico: la tabla ingredients usa nombre, no name.
  • Correccion: insercion por query builder con columna real nombre.

Paso C - Segundo error de columnas en recetas

  • Error real al POST /api/recipes: Unknown column 'name' in recipes.
  • Diagnostico por migracion: la tabla recipes tiene titulo, pasos, foto, time.
  • Correcciones aplicadas:
- RecipeRequest: de name/instructions/photo -> titulo/pasos/foto
- Recipe model $fillable: de name/instructions/photo -> titulo/pasos/foto

Paso D - Error estructural en category_recipe

  • Error real: Unknown column 'category_recipe.recipe_id' al hacer sync.
  • Diagnostico con SHOW COLUMNS: category_recipe tenia solo id, created_at, updated_at.
  • Se creo migracion correctiva para anadir:
recipe_id (FK -> recipes.id)
category_id (FK -> categories.id)
unique(recipe_id, category_id)
  • Nota: se generaron 2 migraciones con mismo nombre y se aplicaron ambas; esto no rompio porque la segunda quedo sin cambios.

Paso E - Ajuste de sync de ingredientes al esquema real

  • Diagnostico: ingredient_recipe no tenia amount/unit, solo IDs.
  • Correccion: ingredients()->sync($ingredientIds) solo por IDs.

Paso F - Pruebas funcionales finales

  • POST /api/recipes exitoso: receta creada con id=2.
  • Verificacion BD:
category_recipe where recipe_id=2 -> [1]
ingredient_recipe where recipe_id=2 -> [1]
  • PATCH /api/recipes/2 exitoso con arrays vacios para relaciones.
  • Verificacion BD tras update:
count(category_recipe where recipe_id=2) = 0
count(ingredient_recipe where recipe_id=2) = 0

Conclusión del bloque: sincronización de relaciones en store/update funcionando.

5) Estado final al cerrar esta sesion

  • CRUD base backend de recetas operativo para create/update/delete con sincronizacion de categorias e ingredientes.
  • Seguridad base activa: policy owner-only, user_id desde auth, validacion en request, use_validated true, batch off.
  • Detalle tecnico a limpiar luego: migracion duplicada de fix category_recipe (ya aplicada). Se puede dejar documentada y no tocar en caliente si no molesta.
  • Siguiente paso planificado: pruebas completas en Postman (store/show/index/update/delete + casos 403).
